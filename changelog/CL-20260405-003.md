## Summary
FEAT04-inbox: implement full Unified Inbox — conversations API with Redis cache, Facebook + LINE webhook handlers, reply endpoint via QStash worker, Pusher real-time UI.

## Changes
- GET /api/conversations — Redis cache (NFR2 < 500ms), version-busted on reply/webhook
- GET /api/conversations/[id] — conversation detail + full message history
- POST /api/conversations/[id]/reply — persist to DB + enqueue QStash send + Pusher event
- workers/send-message — QStash worker calls FB Messenger API or LINE push API (NFR3: 5 retries)
- Facebook webhook — GET verify hub.challenge; POST inbound: upsert customer+conversation+message + Pusher (NFR1 < 200ms async)
- LINE webhook — POST inbound with HMAC-SHA256 verify: upsert customer+conversation+message + Pusher
- conversationRepo — add getMessages(), upsertFacebookCustomer(), upsertLineCustomer()
- Inbox page — wire real API, usePusher hook for real-time new-message, optimistic reply with rollback
- ChatView + ConversationList — add loading prop
