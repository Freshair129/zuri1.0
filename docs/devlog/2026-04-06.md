### Schema — `prisma/schema.prisma`

**`ClassAttendance`** (NEW)

**`Certificate`** (NEW)
- `certificateId String @unique` — format `CERT-YYYYMMDD-NNN` (daily scope per tenant)
- `level`: BASIC_30H / PRO_111H / MASTER_201H (จาก id_standards.yaml)
- `enrollmentId String @unique` — G7: idempotency guard (1 enrollment = 1 cert max)
- `pdfUrl String?`, `notifiedAt DateTime?`

**`CourseSchedule`** (UPDATED)
- เพิ่ม `tenantId String @default("10000000-0000-0000-0000-000000000001") @map("tenant_id")`
- เพิ่ม `attendances ClassAttendance[]`, `@@index([tenantId])`, `@@index([scheduledDate])`
- หมายเหตุ: ใส่ `@default(...)` เพราะ table มี 8 rows อยู่แล้ว — ป้องกัน "required column without default" error ตอน db push

### Repository — `src/lib/repositories/certificateRepo.js` (NEW)

```
generateCertId(tenantId)          → CERT-YYYYMMDD-NNN (daily scope, auto-increment)
getCertLevel(hoursCompleted)      → MASTER_201H | PRO_111H | BASIC_30H | null
findMany(tenantId, opts)          → list certs (pagination + customerId filter)
findById(tenantId, id)            → by UUID or certificateId string
createForEnrollment(tenantId, {}) → idempotent: เช็ค existing ก่อน (G7), return { cert, created }

```

**`GET /api/culinary/certificates`** (NEW)

**`GET /api/culinary/certificates/[id]`** (NEW)

- Body: `{ tenantId }`
- Flow: `verifyQStashSignature` → `findCompletableEnrollments` → loop: `completeEnrollment` + `createForEnrollment` (idempotent)
- `throw error` ที่ท้าย → QStash retry (NFR3 >= 5 retries)

## Session 2 — GitHub Push + Secret Removal

**ปัญหาที่เจอ:**

### GitHub Push Protection บล็อก push
- GitHub ตรวจพบ Obsidian API key (`822fc84dc5bb2f...`) ใน `.mcp.json` ที่ commit เก่า (`c9ebea8`)
- ระบุผิดว่าเป็น "GCP API Key Bound to a Service Account" (false positive)
- Push blocked แม้จะ add `.mcp.json` ไปที่ `.gitignore` แล้ว — เพราะ key ยังอยู่ใน git history

### วิธีแก้: git filter-branch
รัน PowerShell script (`E:\fix_secret.ps1`) บน Windows เพราะ sandbox ไม่มี git credentials:

```powershell
# ลบ .mcp.json ออกจาก history ทุก commit (72 commits ทุก branch)
git filter-branch -f --index-filter `
  'git rm --cached --ignore-unmatch .mcp.json' `
  --prune-empty -- --all
```

ผลลัพธ์: `a639afa...2c08d81 main -> main (forced update)` ✅

### Context สำคัญ
- cmd shell บน Windows `git commit -m "..."` พัง ถ้า message มี slash/parentheses → ต้องใช้ `-F <file>` เสมอ
- `git filter-branch` ใน PowerShell ต้องรันเป็น script file (`.ps1`) ไม่ใช่ inline — quoting ใน cmd/ps1 ต่างกัน
- force push ผ่านแล้ว: `git push origin main --force`
- เพิ่ม `.mcp.json` เข้า `.gitignore` เพื่อป้องกัน commit ในอนาคต

## Session 4 — Fix Vitest (Linux Sandbox Compatibility)

**ทำไปแล้ว:**

### Root Cause Analysis

**Bug 1: `@rolldown/binding-linux-x64-gnu` หาย**
- `npm install` รันบน Windows → ดึงเฉพาะ `binding-win32-x64-msvc`
- Vitest 4.x ใช้ Rolldown (Rust bundler) ต้องการ platform-specific binary
- sandbox = Linux x64 → ต้องการ `binding-linux-x64-gnu`

**Bug 2: `PrismaClientInitializationError` (Unhandled Rejection)**
- `db.ts` ตรวจสอบ `isMockMode = !process.env.DATABASE_URL` → false (มี DATABASE_URL ใน setup)
- → `new PrismaClient()` ถูกเรียก → พยายาม load Windows `.query-engine` binary บน Linux → Error
- แม้ tests pass หมด แต่ unhandled rejection ทำให้ exit code = 1

### Fixes

**Fix 1:** install Linux binary (coexist กับ Windows — ไม่กระทบ production):
```bash
npm install @rolldown/binding-linux-x64-gnu@1.0.0-rc.12 --no-save
```

**Fix 2:** เพิ่มใน `src/tests/setup.js`:
```js
process.env.MOCK_MODE = 'true'  // force isMockMode=true → PrismaClient ไม่ถูกเรียก
```

**Fix 3:** เพิ่มใน `prisma/schema.prisma` generator:
```prisma
binaryTargets = ["native", "windows", "debian-openssl-3.0.x"]
```
(สำหรับ `prisma generate` ในอนาคตเมื่อย้าย node_modules มา Linux)

### Context สำคัญ
