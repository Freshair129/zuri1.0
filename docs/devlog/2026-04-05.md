### core-auth Implementation (4 ไฟล์)

**`src/app/(auth)/login/page.jsx`** — Static UI → Functional form
- เพิ่ม `useState` (identity, password, error, loading, showPassword)
- Wire form submission กับ `signIn('credentials', { redirect: false })`
- Error banner แสดงเมื่อ credentials ผิด
- Loading state disable button + เปลี่ยน label
- Show/hide password toggle
- Link จริงไปยัง `/forgot-password` และ `/register` ด้วย Next.js `<Link>`
- รับ `callbackUrl` จาก searchParams → redirect หลัง login สำเร็จ

**`src/lib/auth.js`** — เพิ่ม RBAC helpers
- `requireRole(domain, action)` — check RBAC ก่อน return session, throw 403 ถ้า forbidden
- `withAuth(handler, { domain, action })` — HOC สำหรับ API routes, return 401/403 JSON อัตโนมัติ
- `normalizeRole(role)` — map legacy roles (MGR→MANAGER, SLS→SALES, etc.) ตาม ADR-068

**`src/app/api/auth/[...nextauth]/route.js`** — Legacy role migration
- `authorize()` เรียก `normalizeRole()` บน `employee.roles[]` ก่อนเก็บลง JWT
- Support ทั้ง `employee.roles` (array) และ `employee.role` (string fallback)
- เพิ่ม `try/catch` + `console.error('[NextAuth.authorize]', err)`

**`src/middleware.js`** — ปรับปรุง auth guard
- `PUBLIC_PATHS` set สำหรับ `/login`, `/register`, `/forgot-password`
- `SKIP_PREFIXES` array สำหรับ `/_next`, `/api/auth`, etc.
- Redirect authenticated users ออกจาก auth pages → `/`
- Redirect unauthenticated users พร้อม `?callbackUrl=pathname`
- Default `tenantId = '10000000-0000-0000-0000-000000000001'` ถ้า JWT ไม่มี
