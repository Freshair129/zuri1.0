# Hello World Verification

## 1. Summary
A foundational feature to verify the end-to-end connectivity of the Zuri Platform v2 architecture. It provides a simple endpoint and UI component to ensure the Next.js App Router, API handlers, and basic styling are functioning as expected.

## 2. User Stories
- As a **Developer (DEV)**, I want to access a "Hello World" page, so that I can confirm the deployment and routing are working correctly.
- As a **Technician (TEC)**, I want to trigger a "Hello World" API, so that I can verify the backend connectivity and response format.

## 3. Data Flow
```
[User visits /hello] → [App Router] → [Hello Component]
                                            ↓
[Component Mount] → [GET /api/hello] → [API Route Handler]
                                            ↓
[UI Update] ← [JSON Response: { message: "Hello World" }]
```

## 4. API Endpoints
| Method | Path | Purpose | Auth |
|--------|------|---------|------|
| GET | /api/hello | Returns a simple hello world JSON response | GUEST (Public) |

## 5. Database Changes
- [x] No database changes required for this verification feature.

**ADR Required:** No

## 6. Roles & Permissions
| Role | Access |
|------|--------|
| ALL | View "Hello World" page and call API |

## 7. UI Components
- [ ] `HelloWorld.jsx` — A simple card component displaying the message fetched from the API.
- [ ] `app/hello/page.jsx` — The page route hosting the component.

## 8. Edge Cases & Gotchas
- **Network Latency:** Ensure the UI handles a loading state while fetching from `/api/hello`.
- **Environment Variables:** If the message were to come from a config, ensure `.env` is loaded (not applicable for this simple version).

## 9. Acceptance Criteria
- [ ] Navigating to `/hello` displays a "Hello World" message.
- [ ] Calling `GET /api/hello` returns `200 OK` with `{ "message": "Hello World from Zuri v2" }`.
- [ ] The UI follows the TailwindCSS utility patterns established in the project.

## 10. Dependencies
- Depends on: Next.js App Router Setup.
- External: None.
