## 1. Overview

Feature นี้เป็น **internal test harness** สำหรับตรวจสอบว่า co-dev pipeline สามารถ execute tool-calling loop ได้อย่างถูกต้องและสมบูรณ์ — ครอบคลุมการ retrieve context, call LLM, parse tool calls, execute tools, และส่ง result กลับเข้า loop โดยไม่ truncate หรือ hang

## 2. Terminology

| คำ | นิยาม |
|---|---|
| **Tool Calling Loop** | วงจร: LLM ออก tool call → executor รัน tool → ผล feed กลับ LLM → วนซ้ำจนได้ final answer |
| **Antigravity** | co-dev executor engine (`.dev/co-dev/core/pipeline.py`) |
| **Gate** | ขั้นตอน validation ก่อน/หลัง tool execution (GateBlockedError) |
| **Turn** | 1 รอบ LLM call + tool execution |
| **Max Turns** | จำนวนรอบสูงสุดก่อน force-stop (ป้องกัน infinite loop) |

## 3. Feature Breakdown

- As a **DEV**, I want to trigger a test tool-calling sequence, so that I can verify the pipeline handles multi-turn tool calls correctly.
- As a **DEV**, I want the loop to stop gracefully after `max_turns`, so that runaway loops don't block the system.
- As a **DEV**, I want each tool result injected back into the LLM context correctly, so that tool outputs are not lost between turns.
- As a **DEV**, I want truncation detection to fire if `pipeline.py` or `llm.py` output is cut short, so that G-TRUNC-01 is enforced automatically.

## 4. Data Flow

```
[Test Trigger]
      │
      ▼
[pipeline.py: run_turn()]
      │
      ├──► [llm.py: call_gemini()] ──► [Gemini API]
      │           │
      │           ▼ ToolCall response
      ├──► [tools.py: execute(tool_name, args)]
      │           │
      │           ▼ ToolResult
      ├──► [inject result → messages[]]
      │
      ▼ (loop until no tool calls OR max_turns)
[Final LLM text response]
      │
      ▼
[pipeline.py: return result]
```

## 5. Roles & Permissions

| OWNER | Read logs only |
| Others | No access |

## 6. NFR

- Response time: ≤ 30s per full loop (5 turns × 6s max per LLM call)
- Retry policy: QStash retry ≥ 5 ครั้ง ถ้า pipeline crash
- Truncation guard: ตรวจ `wc -l pipeline.py ≥ 320` และ `tail -20` หลังแก้ไขทุกครั้ง (G-TRUNC-01)
- Max turns hard limit: 10 turns — GateBlockedError หลังเกิน

## 7. Known Gotchas

- **G-TRUNC-01**: Antigravity truncate `pipeline.py`/`llm.py` ซ้ำที่ช่วง LLM call — ตรวจ `tail -20` เสมอ (ดู `docs/gotchas/codev-truncation.md`)
- Tool result ต้อง serialize เป็น string ก่อน inject กลับ — Gemini ไม่รับ object โดยตรง
- `GateBlockedError` ต้อง defined ที่ท้ายไฟล์ `pipeline.py` — truncation มักตัดตรงนี้

## 8. Implementation Phases

- [ ] Phase 1: เขียน test script `scripts/test-tool-loop.py` — trigger 3-turn loop ด้วย mock tools
- [ ] Phase 2: เพิ่ม truncation self-check ใน `pipeline.py` (`assert len(lines) >= 320`)
- [ ] Phase 3: integrate กับ CI — รัน syntax check + wc-l ทุกครั้งที่แก้ core files

## 9. Related

- ADR: ADR-068 (RBAC — gate access control)
- Spec: `.dev/co-dev/core/pipeline.py`, `.dev/co-dev/core/llm.py`
- Docs: `docs/gotchas/codev-truncation.md`, `docs/handoff/IMP-20260404-codev-context-injection.md`

## 10. Acceptance Criteria

- [ ] Loop executes 3 turns ด้วย mock tools โดยไม่ error
- [ ] Tool results ปรากฏใน final LLM context ถูกต้อง
- [ ] Loop หยุดเมื่อ `max_turns` ถึงและ raise `GateBlockedError`
- [ ] `wc -l pipeline.py ≥ 320` และ `tail -20` แสดง `GateBlockedError` class หลังแก้ไข
- [ ] `python -m py_compile pipeline.py` pass ทุกครั้ง

### Issues Found
None critical. Minor observations:

1. **NFR mismatch**: Spec says "QStash retry ≥ 5 ถ้า pipeline crash" but this is a local test harness script — QStash retry applies to deployed workers, not local test scripts. Clarify whether this test will also run as a QStash worker or is purely local CLI.

2. **`call_with_tools` signature**: G-TRUNC-01 retrieved context (chunk [1]) shows the correct keyword args. Ensure the test script uses `tool_executor=executor.execute` (callable), not `executor=executor` (object). This is the #1 Antigravity mistake.

3. **Missing section 9 detail**: "Related" section references ADR-068 (RBAC) but this feature has no RBAC dependency — it's a DEV-only test harness. The link is misleading. Better to reference ADR for the pipeline itself if one exists, or mark "None".

### ADR Required
None — this is an internal test harness with no new dependencies, no schema changes, and no architectural decisions.

### Notes for Implementation
- Use the **correct `call_with_tools` signature** per `docs/gotchas/codev-truncation.md` — `tool_executor=executor.execute`, not `executor=executor`
- After writing/modifying `pipeline.py` or `llm.py`, always run the G-TRUNC-01 triple check: `wc -l` ≥ 320, `tail -20` shows `GateBlockedError`, `python -m py_compile` passes
- Mock tools should return **string results** (not dicts/objects) — Gemini API requires string tool outputs
- The `max_turns=10` hard limit with `GateBlockedError` is correct defensive design
- Test script location `scripts/test-tool-loop.py` is appropriate — keeps it outside core
