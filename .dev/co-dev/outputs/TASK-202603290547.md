# Feature Spec: AI Compose Reply

**Version:** 1.0.0
**Date:** March 29, 2026
**Status:** DRAFT
**Author:** PM Agent

---

## 1. Overview
The AI Compose Reply feature leverages Gemini 2.0 Flash to generate context-aware response suggestions for sales agents within the Unified Inbox. It aims to reduce response time and maintain a professional, personalized brand voice by analyzing chat history and customer profile data.

## 2. User Stories
- As a **Sales Agent (AGT)**, I want to click an "AI Compose" button in the chat box so that I can get a drafted reply based on the customer's last question without typing it myself.
- As a **Marketing Manager (MKT)**, I want the AI to know the customer's membership tier and enrollment history so that the generated replies can upsell relevant courses or acknowledge their loyalty.
- As a **System Administrator (DEV)**, I want to ensure every AI request is scoped to a `tenant_id` so that private school data never leaks between different clients.

## 3. Acceptance Criteria
- [ ] AI must generate a reply suggestion in under **2.0 seconds** (NFR1 compliance).
- [ ] Suggestions must be inserted into the `ReplyBox` but **not sent automatically**; human review is mandatory.
- [ ] The prompt must include the last **10 messages** of conversation history for context.
- [ ] The system must pull `Customer.intelligence` and `Customer.tier` to personalize the greeting and offer.
- [ ] A "Re-generate" button must be available to provide an alternative phrasing if the first one is unsuitable.

## 4. Data Flow
1. **User Action:** Agent clicks the "AI Compose" icon in `src/modules/core/inbox/components/ReplyBox.jsx`.
2. **Context Assembly:** Frontend sends `conversationId` and `tenantId` to the backend.
3. **API Logic:** `POST /api/core/ai/compose-reply` is triggered.
4. **Data Retrieval:** Backend fetches conversation history from `core-inbox.prisma` and customer metadata from `core-crm.prisma` (filtering by `tenant_id`).
5. **AI Request:** Backend calls `src/modules/core/ai/geminiService.js` with a structured prompt (System Role + Context + History).
6. **Processing:** Gemini 2.0 Flash generates the response string.
7. **Response:** API returns JSON payload `{ "suggestion": "..." }` to the client.
8. **UI Update:** The `ReplyBox` text area is populated with the suggestion, highlighted for review.

## 5. API Endpoints Required
| Method | Path | Auth | Roles | Description |
|--------|------|------|-------|-------------|
| POST   | /api/core/ai/compose-reply | JWT | SLS, AGT, MKT | Generates a reply suggestion for a specific conversation. |
| GET    | /api/core/ai/settings | JWT | MGR, DEV | Retrieves tenant-specific tone-of-voice settings. |

## 6. Database Changes
- **New tables:** None.
- **Modified tables:** 
    - `Customer`: `intelligence` (JSONB) — Update this field periodically with summarized intent discovered during AI generation.
- **New indexes:** None.
- **Schema notes:** Ensure `tenant_id` is indexed on both `Conversation` and `Message` tables for rapid context retrieval.

## 7. Roles & Permissions
| Action | Allowed Roles |
|--------|--------------|
| Use AI Compose | SLS, AGT, MKT, MGR, DEV |
| Modify System Prompt/Tone | MGR, DEV |
| View AI Token Usage | MGR, ACC, DEV |

## 8. UI Components Required
- `AIComposeButton.jsx` — Integrated into `src/modules/core/inbox/components/ReplyBox.jsx`.
- `AIResponsePreview.jsx` — A subtle overlay showing the generated text before it populates the main input.
- `ToneSelector.jsx` — Small dropdown to switch between "Formal", "Friendly", or "Sales-driven" tones.

## 9. Tech Notes & Gotchas
- **Rate Limiting:** Implement a 10-requests-per-minute limit per agent to manage Gemini API costs.
- **Hallucinations:** AI might invent course dates or prices. UI must display a "Verify info before sending" warning (ADR-048).
- **Privacy:** Ensure no PII (Personally Identifiable Information) beyond Name is sent to the LLM unless necessary for the prompt.
- **Multi-language:** The system must detect if the conversation is in Thai or English and reply in the same language.

## 10. Out of Scope
- **Auto-Sending:** The AI will never send a message without a human clicking "Send".
- **Bulk Messaging:** This feature is for 1-on-1 chat only; not for broadcasting to multiple customers.
- **Voice-to-Text:** Converting customer voice notes into text is handled by a separate "Transcribe" feature.
